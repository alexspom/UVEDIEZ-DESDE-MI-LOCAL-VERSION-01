Mantenimiento de recursos

Desde esta pantalla se mantiene todo lo referente a los recursos con los que cuenta la aplicación para realizar las tareas necesarias, puede ser desde un Terminal de mano hasta un montacargas etc.

Consta de dos bloques, el mantenimiento propiamente dicho y las tareas que tiene asignadas cada recurso

Para poder realizar cualquier operación de tipo alta, baja o modificación, se ha de tener el privilegio de “IMPLANTADOR”.

Las teclas de función utilizadas en esta pantalla son:

{F1} Ayuda {F4} Confirmar Cambios {F6} Cambio de pantalla

{F9} Borrar Registro {May-F10} Menú contextual

El menú contextual de esta pantalla es VDRECURSO

Accede directamente a la tabla VDRECURSO.

Los campos de este bloque son:

  • Código: Es el código que identifica al recurso. A continuación se muestra su descripción

  • Activo: Switch que indica si el recurso está o no activo para que la aplicación le utilice. Sus valores posibles son ’o ’.

  • Dirección: Se refiere a la dirección física del recurso, normalmente la dirección IP.

  • Conectado: indica si el recurso está o no conectado. Sus valores posibles son ’o ’.

  • Sid: Es el sid de la sesión de oracle a la que está conectado el recurso.

  • Serial: Es el serial de la sesión de oracle a la que está conectado el recurso.

  • Terminal: Máquina a la que se ha conectado el recurso.

  • Contenedores: Indica si el recurso puede o no transportar contenedores. Sus valores posibles son ’o ’.

  • Embalajes: Indica si el recurso puede o no transportar embalajes. Sus valores posibles son ’o ’.

  • Unidades: Indica si el recurso puede o no transportar unidades. Sus valores posibles son ’o ’.

  • Máxima altura: Es la máxima altura a la que podrá acceder el recurso.

  • Mínima: Es la mínima altura a la que podrá acceder el recurso.

  • Admite teclear: Indica si el recurso puede teclear campos que se pueden leer mediante escáner. Sus valores posibles son ’o ’.

  • Ubicación asignada: Es la ubicación temporal asignada al recurso y que se utiliza mientras transporta de un sitio a otro.

  • Matrícula: Es la matrícula asignada al recurso.

  • Usuario: Es el operario identificado en el recurso que es un Terminal.

  • Priv. Login: Es el privilegio necesario para acceder a este recurso.

  • Estado: Es el estado del recurso. A continuación aparece la descripción de dicho estado.

  • Driver: Es el driver C que gestiona el recurso.

  • Perfil: Es el perfil que está ejecutando el recurso.

  • Pantalla: Nombre de la pantalla que ejecuta el recurso.

  • Impresora: Es el código nemotécnico de la etiquetadora/impresora

  • Planificador lib: Librería dónde está ubicado el planificador del recurso.

  • Planificador Func: Función que usa el planificador del recurso.

  • Planificador param: Parámetros que se pasan al planificador del recurso.

  • Extra: Información extra para el implantador de V10.

  • Comentario: Es simplemente un comentario a libre disposición de uso.

  • Última ubicación visitada: Es la última ubicación en la que estuvo el recurso.

  • Última modificación: Consta de tres campos, el primero almacena quién realiza la modificación, bien sea un usuario o un procedimiento, el segundo y el tercero almacenan la fecha y la hora respectivamente, en la que se realizó la última modificación.
