Privilegios por defecto en pantallas

Desde esta pantalla se definen los privilegios por defecto en los bloques de datos de las pantallas, en cuento a poder realizar operaciones con los datos mostrados.

Para realizar cualquier modificación de tipo alta, baja o modificación, se necesita poseer el privilegio de “IMPLANTADOR”

 

Consta de dos bloques, que acceden directamente a la tabla VDDEFBLOQUE:

El único campo que muestra es el grupo del bloque, es decir si es referente a Artículos, Bultos, etc.

 

El segundo bloque contiene los campos:

 

  • Pantalla: Es el nombre de la pantalla o bloque.

  • Grupo: Grupo de pantallas al que pertenece el bloque

  • T: Es el tipo de pantallas a las que se aplicarán los defectos. Sus valores son: (F)orms, (R)adios o (A)mbos.

  • PrivInsert: Muestra el privilegio necesario para insertar registros.

  • PrivDelete: Muestra el privilegio necesario para borrar registros.

  • PrivNoenter: Muestra el privilegio necesario para insertar registros.

  • PrivNoupdate: Muestra el privilegio necesario para modificar registros.

  • PrivNoorains: Muestra el privilegio necesario para insertar registros.

  • Última Modificación: Consta de tres campos, el primero almacena quién realiza la modificación, bien sea un usuario o un procedimiento, el segundo y el tercero almacenan la fecha y la hora respectivamente, en la que se realizó la última modificación.
