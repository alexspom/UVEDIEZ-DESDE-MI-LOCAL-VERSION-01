Mantenimiento de usuarios

Desde esta pantalla vamos a poder definir lo referente a los usuarios, desde los usuarios en sí, a las asignaciones de estos a los diferentes grupos, así como a los diferentes privilegios.

Se compone de tres bloques, y las teclas de función no son las mismas para los tres, por lo que se describirán dentro de cada uno.

Con el privilegio de “SUPERUSUARIO”, las operaciones que se pueden realizar desde los bloques son altas, bajas y modificaciones.

A continuación describiremos cada bloque:

- Mantenimiento de Usuarios

Accede directamente a la tabla VDUSUARIO.

Para dar un alta, será necesario copiar un registro ya existente y modificarlo.

Sus campos son los siguientes:

  • Código: En este campo definiremos el En este campo definiremos el código del operario.

  • Nombre: En este campo definiremos el nombre del operario.

  • Apellido1: Indicaremos el primer apellido del usuario.

  • Apellido2: Indicaremos el segundo apellido del usuario.

  • Iniciales: Nos indicara las iniciales del usuario.

  • Imprimir: Switch que indica si se quiere imprimir la etiqueta del usuario seleccionado. Si su valor es ’al pulsar F4 se imprimirá dicha etiqueta.

  • Tema: Indica el tema de colores de la aplicación, activo para el usuario seleccionado.

  • N.: Nos indicará el nivel de privilegio que tendrá el usuario en su menú.

  • B.: Nos indica el bloqueo de cuenta del operario. Los posibles valores son:

      • N: No bloqueado.

      • S: Bloqueado por administrador.

      • R: Rechazado por demasiados errores en login.

      • C: Caducado.

  • Menú Asociado: Nos indica el menú que asociamos al usuario.

  • %Ver: Es el porcentaje de bultos servidos por este usuario, que se deben verificar.

  • Er.: Contador de errores a la hora de logarse para entrar en la aplicación.

  • Err. T: Contador total de errores a la hora de logarse para entrar en la aplicación.

  • Logins: Cuenta las veces que se ha logado correctamente.

  • Caducidad: Fecha en la que caduca la contraseña.

  • Alta: Fecha y hora en la que se ha dado de alta a ese usuario.

- Password: Hash del password del usuario; es decir el resultado de una función o algoritmo para generar claves o llaves que representen de manera casi unívoca a un documento, registro, archivo, etc.

- Clave pública: Clave pública del usuario, es utilizada para la firma digital de documentos o informes o modificaciones a registros, es decir la firma de la operación. Es conocida por todos los usuarios implicados en la operación y necesaria para verificar la autenticidad del documento.

- Clave privada: Clave privada del usuario, es utilizada para la firma digital para firmar el documento y cifrar su contenido, siendo únicamente conocida por el usuario que firma el documento.

- Última modificación: Consta de tres campos, el primero almacena quién realiza la modificación, bien sea un usuario o un procedimiento, el segundo y el tercero almacenan la fecha y la hora respectivamente, en la que se realizó la última modificación

 

Las teclas de función en este bloque son:

{F2} Consulta {F4} Confirmar cambios {F5} Borrar pantalla

{Ctrl-F7} Copia {May+F10} Menú contextual

El menú contextual de esta pantalla es VDUSUARIO

Los otros dos bloques son:

  • Asignación de usuario a grupos

  • Asignación de privilegios a usuario
